# 1.1 声音的数字化与时域波形

在计算机世界里，声音是如何被捕捉并存储的？本小节将为您介绍音频数字化的核心概念及其在时域上的波形呈现方式。

---

## 1. 声音的物理本质
声音在物理学中是介质中传播的**机械波（纵波）**，是由于物体振动引起周围空气密度交替变化（疏密波）而产生的压力变化。
我们听到的各种声音，本质上都是空气压强随时间变化的连续曲线。

---

## 2. 模拟信号与数字信号
*   **模拟信号（Analog Signal）**：自然界中的声波是连续的模拟信号，时间和幅度都是无限连续的。
*   **数字信号（Digital Signal）**：计算机只能处理离散的 0 和 1。为了在计算机中处理声音，必须将连续的模拟声波转换为离散的数字序列。

---

## 3. 音频数字化的三大要素 (ADC 转换)
将模拟信号转换为数字信号的过程称为 **模数转换 (Analog-to-Digital Conversion, ADC)**。其核心包括三个核心指标：

### 3.1 采样率 (Sampling Rate)
*   **定义**：每秒钟对声波进行采样的次数，单位为赫兹 (Hz)。
*   **奈奎斯特采样定律 (Nyquist Theorem)**：为了不失真地重建原始信号，采样率必须大于原始信号最高频率的 **2 倍**。
*   **常用标准**：
    *   **8,000 Hz (8kHz)**：电话语音采样率（能满足基本人声沟通）。
    *   **16,000 Hz (16kHz)**：**语音 AI 领域的行业标准**（SenseVoice、Kokoro、Whisper 等模型默认接收 16kHz 音频，在保证音质的同时减少计算开销）。
    *   **44,100 Hz (44.1kHz)**：CD 级无损音质（人耳听觉极限约 20kHz）。

### 3.2 采样位数 / 位深 (Bit Depth)
*   **定义**：记录每次采样幅值时所使用的二进制位数，决定了声音的动态范围和量化精度。
*   **常见位深**：
    *   **8-bit**：量化级为 $2^8 = 256$，噪底高。
    *   **16-bit (Int16)**：**最常用的音频存储格式**，量化级为 $2^{16} = 65,536$，动态范围约为 96dB。
    *   **32-bit Float**：浮点型，幅值归一化在 `[-1.0, 1.0]` 之间，常用于数字音频工作站 (DAW) 和深度学习模型的输入。

### 3.3 声道数 (Channels)
*   **单声道 (Mono)**：语音模型（ASR/TTS/VAD）输入通常硬性要求**单声道**音频。
*   **双声道/立体声 (Stereo)**：包含左右两个独立的声道。

---

## 4. 时域波形 (Time-Domain Waveform)
在数学上，数字化后的声音可以表示为一个一维数组：
$$X = [x_1, x_2, x_3, \dots, x_N]$$
每个元素 $x_i$ 代表特定时间点的空气压强偏差。

将这个一维数组作为纵轴，时间作为横轴绘制出来的图形，就是我们常说的**时域波形图**。
*   **优点**：非常直观地展现声音在时间轴上的**能量变化**、起伏和静音区间（这也是 VAD 算法的工作基础）。
*   **缺点**：无法直观地展示声音的**音调高低（频率特征）**。同样一段能量的波，我们无法仅凭眼睛看出它是高音还是低音。为了解决这个问题，我们需要进入“频域”。
