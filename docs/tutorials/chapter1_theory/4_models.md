# 1.4 ASR/TTS 声学模型与声码器工作原理

在了解了语音数字信号的底层表示后，本节将向您揭示 ASR（语音识别）和 TTS（语音合成）两大应用层模型是如何处理这些数据的。

---

## 1. 语音识别 (ASR) 的网络模型架构
语音识别的终极目标是将二维的 Fbank 特征矩阵 $X$ 转写为文本词序列 $Y$。目前主流模型均采用端到端深度学习结构：

### 1.1 编码器 (Encoder)
ASR 模型的输入通常是一帧帧拼接起来的 Fbank 频谱（例如大小为 $T \times 80$ 的特征图）。
*   **特征提取**：首先通过 2D 卷积网络提取图像般的频谱时空局部相关特征。
*   **上下文感知**：利用 Transformer 或 Conformer 结构，学习长序列中各发音节点前后的依赖关系，以确定音素边界。

### 1.2 对齐与输出解码 (Alignment & Decoder)
因为每一帧特征（约 10ms）的时间粒度远小于一个汉字发音的时间（约 200ms），模型必须进行“时间轴对齐”。常见的对齐解码技术有：
*   **CTC (Connectionist Temporal Classification)**：引入一个空字符标签 $\epsilon$，通过动态规划自动对齐。SenseVoice 等超快速 ASR 大多在首层结合了 CTC 进行加速预测。
*   **AED (Attention-based Encoder-Decoder)**：利用交叉注意力机制进行自回归解码，生成文字更精准。

---

## 2. 语音合成 (TTS) 的工作机制
语音合成（Text-to-Speech）的流程刚好与 ASR 相反，它的目标是将文本序列 $Y$ 合成为波形音频数据 $X$。现代高保真 TTS 普遍采用两阶段架构：

```
[文本输入] ───> ( 1. 声学模型 ) ───> [中间特征: 梅尔频谱] ───> ( 2. 声码器 ) ───> [声音波形 (WAV)]
```

### 2.1 第一阶段：声学模型 (Acoustic Model)
*   **任务**：将输入的文本词汇序列（或拼音/音素序列）映射并翻译为声学维度的**梅尔频谱图 (Mel-Spectrogram)**。
*   **代表模型**：FastSpeech、VITS、Qwen-TTS、Kokoro-82M（声学阶段输出梅尔谱特征）。

### 2.2 第二阶段：声码器 (Vocoder)
*   **任务**：梅尔频谱图仅仅是频率幅度的分布，丢失了波形相位信息，并且不是能够直接被扬声器播放的时域声波。**声码器的唯一职责，就是将“梅尔谱特征”重新合成为“时域声音波形信号（WAV 字节流）”**。
*   **代表模型**：HiFi-GAN、BigVGAN。声码器的合成速度和保真度直接决定了生成语音是否带有冰冷的机器音、电子音杂音，是 TTS 情感传达的关键。
