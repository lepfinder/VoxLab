# 2.1 准备工作与硬件选型

欢迎来到硬件实战篇。本小节将为您介绍在开始开发 ESP32 语音对讲端之前所需的硬件准备、选型建议和整体架构设计。

---

## 1. 为什么选择 ESP32 作为语音对讲设备？
*   **高性价比**：一块标准的 ESP32 开发板仅需十几元人民币。
*   **无线连接支持**：内置 2.4GHz Wi-Fi 和蓝牙，能够摆脱线缆束缚，支持通过 IP 网络远程传输语音数据。
*   **丰富的外设接口**：支持硬件 **I2S (Inter-IC Sound)** 总线接口，这是连接高性能数字麦克风与音频编解码（Codec）芯片的工业标准。
*   **算力强劲**：双核 Tensilica LX6/LX7 处理器，主频高达 240MHz，拥有足够算力在本地运行软硬件音频处理和编码（如 Opus 压缩）。

---

## 2. 硬件清单与推荐选型

要构建一个完整的语音对讲输入端，您需要以下核心配件：

### 2.1 主控芯片
*   **推荐型号**：**ESP32-S3-WROOM-1** 开发板（推荐带有 **8MB PSRAM** 的版本）。
*   *原因*：语音处理在分帧、量化和 Opus 编码阶段对 RAM 开销极大，ESP32-S3 内置的向量指令集（Vector Extension）能加速数字信号处理，且 PSRAM 能防止内存溢出。

### 2.2 数字麦克风 (音频输入)
*   **推荐型号**：**INMP441** 数字麦克风模块 或 **MSM261S4030H0** 麦克风。
*   *连接方式*：**I2S 接口**。
*   *特点*：数字麦克风内置了前置放大器和 ADC（模数转换器），直接输出离散的 24-bit/16-bit PCM 数字信号，相比于传统的模拟麦克风，抗电磁干扰和抗噪性能极佳。

## 3. 系统拓扑架构图
在语音实验室的实时对讲链路中，数据的流向设计如下：

```
 [ INMP441 麦克风 ]  ───(I2S 单声道/16k/16bit)───> [ ESP32-S3 主控 ]
                                                       │
                                                 (软/硬 Opus 编码)
                                                       │
                                                       ▼
 [ 本地 VoxLab 服务端 ] <──(WS 二进制帧/Opus包)── [ WebSockets 协议 ]
```
